Asset versioning

Setting the version of an asset

Each versioned asset has a version property.

When to create a new version

A new version has a least to be created, when an asset is to be changed but the most recent version has already been approved by SOA governance. Create a new minor version when the changes are backward compatible. However, if the changes are incompatible, a new major version must be created.

Versioned relationships between assets

Relationships between versioned assets are version as well. This is expressed by defined a version constraint rule on the relationship like this:

Example 2.1. 

someAttribute : SomeOtherType majorVersion 2

The example declares that the BO attribute someAttribute references an instance of the latest minor version in mojor version 2 available for VersionedType SomeOtherType.
