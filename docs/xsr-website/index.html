Why XKonneX Repository?

Know your (micro) services

Whether you are targeting developers or domain experts, you will be find, model and understand the structure, meaning and use of services, domain objects and technical assets in your IT infrastructure. Be able to find and define the pieces that make up your software.

Features

Xtext based editors

Model your services with state of the art editors featuring code completion, refactoring, validation etc. The model supports domain driven design, full blown SOA, or lean micro service modeling.

Find and relate

Use specific search features. Relate services, domain objects consumers and other assets using tags.

Graphical views

XKonneX repository provides graphical viewer for your model

Generate code

Generate interface code from your model, be it WSDLs and XSDs, RAML and JSON schema for REST or Java.

Govern your domains and modules

Manage available versions and the lifecycle of your services. Govern them in a SOA style or a lean style. Supports version evolution. But if you don't need it you can avoid it.
